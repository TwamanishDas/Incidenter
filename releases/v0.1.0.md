# Release Notes - v0.1.0

## Release Type

- MVP Phase 1 baseline release (Data Ingestion)
- Status: Ready for tagging

## Scope

This release freezes the Layer 1 ingestion baseline using sample-blob replay mode, with scheduler orchestration, diagnostics, and acceptance validation.

## Included Work

### Phase 1 Steps

1. Step 1: Sample data staged in blob container.
2. Step 2: Switchable ingestion mode implemented (`sample_blob` / `live`).
3. Step 3: End-to-end sample replay validation completed.
4. Step 4: Hardening completed (replay diagnostics + scheduler switch-behavior tests).
5. Step 5: Ingestion health checklist endpoint completed (`GET /ingestion/checklist`).
6. Step 6: Acceptance run + baseline freeze completed.

## Validation Evidence

- Acceptance baseline document:
  - `STEP6_MVP_INGESTION_BASELINE.md`
- Acceptance artifact:
  - `artifacts/step6_acceptance_latest.json`
- Test suite:
  - `.\.venv\Scripts\python.exe -m unittest discover -s backend/tests -p "test_*.py" -v`
  - Current result at release time: all tests passing.

## Key Technical Additions

- Sample replay diagnostics in collector status:
  - `last_blob_name`, list/download status, replay outcome, replay error.
- Ingestion checklist evaluator and endpoint:
  - pass/warn/fail classification for scheduler + collectors + event flow + dedup consistency.
- Acceptance evaluation framework and executable runner.

## Known Limitations

1. Current RCA logic remains threshold/rule based (not full causal RCA).
2. Storage remains in-memory for incidents (no persistent database yet).
3. Sample-blob mode is temporary and intended to be replaced with live Azure ingestion.
4. Stream/network instability can affect long-running doc lookups in chat sessions.

## Out of Scope for v0.1.0

1. Production deployment.
2. Full dashboard and notification workflows.
3. Advanced causal dependency RCA and probability scoring refinement.
